🔐 COMO FUNCIONA AGORA:
═══════════════════════════════════════════════════════════════════════════

1️⃣  CADASTRO:
   User → "Cadastrar" → auth_service.criar_usuario()
   → supabase.auth.sign_up({email, password, data: {nome}})
   → Supabase: insere em auth.users + user_metadata
   → Email confirmação enviado

2️⃣  LOGIN:
   User → "Entrar" → auth_service.fazer_login()
   → supabase.auth.sign_in_with_password({email, password})
   → Supabase: valida + gera JWT token
   → Retorna: {user, session.access_token}
   → Token enviado em Authorization header automaticamente

3️⃣  REQUISIÇÕES AO BANCO:
   repositório.listar() 
   → supabase.from_('tabela').select()
   → Token enviado automaticamente
   → RLS valida: WHERE user_id = auth.uid()
   → Apenas dados do usuário retornados

4️⃣  LOGOUT:
   User → Ícone de logout → on_logout()
   → auth_service.fazer_logout()
   → supabase.auth.sign_out()
   → Sessão invalidada
   → Redireciona para /login

🚨 POSSÍVEIS PROBLEMAS:
═══════════════════════════════════════════════════════════════════════════

⚠️  "Sistema de autenticação não configurado"
   → Falta arquivo .env ou credenciais inválidas
   → Solução: Criar .env com SUPABASE_URL e SUPABASE_ANON_KEY

⚠️  "Email ou senha incorretos"
   → Email não foi confirmado ou credenciais erradas
   → Solução: Confira email/senha, ou use novo email

⚠️  "Email já cadastrado"
   → Email já existe no Supabase Auth
   → Solução: Use outro email (cada email é único)

⚠️  Sessão expira após 1 hora
   → Comportamento normal (JWT expira)
   → Futuro: Implementar refresh_token automático

⚠️  Tabela usuarios não foi removida
   → Se existir banco local antigo
   → Solução: Deletar granja.db e deixar criar novo
